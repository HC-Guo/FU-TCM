verl Docker 训练 Qwen3.5 GRPO 环境与启动指南

目标：把你本地已经微调好的 Qwen3.5 模型、grpodata/verl 数据、 reward_functions.py 和 zhenghou_btree.json 上传到服务器后， 用 Docker 进入 verl 环境进行 GRPO 训练。

本指南默认 NVIDIA GPU、Linux 服务器、FSDP 训练、vLLM rollout。服务器路径会变，所以所有路径都用变量配置。

0. 关键结论

  • 官方 Docker 镜像优先用 verlai/verl:vllm011.latest，因为我们要用 vLLM rollout。
  • 你的本地 grpodata/verl 目前是 .jsonl，verl 训练前要转成 .parquet。
  • reward_functions.py 里已经有 compute_score，verl 参数写 custom_reward_function.name=compute_score。
  • reward_functions.py 会读取同目录下的 zhenghou_btree.json，上传时必须一起传。
  • 先跑 smoke 配置确认链路，再扩大 batch、response length、rollout_n 和 epoch。
官方安装文档写明 verl 需要 Python >= 3.10、CUDA >= 12.8，并推荐 Docker；预构建镜像在 DockerHub verlai/verl，示例包括 verlai/verl:vllm011.latest。数据准备文档要求训练数据保存为 parquet。

1. 服务器目录规划

假设你上传到服务器后使用下面的目录。你可以改，只要后面的变量同步即可。

变量服务器宿主机路径示例内容
PROJECT_DIR /data/tcm_final 项目文件，至少包含 grpodata/verl、reward_functions.py、zhenghou_btree.json
MODEL_DIR /data/models/qwen35_tcm_sft 你在 Qwen3.5 基础上已经微调好的模型目录，里面应有 config.json、tokenizer、权重文件
OUTPUT_DIR /data/outputs/tcm_grpo GRPO checkpoint、日志输出目录
VERL_SRC_DIR /data/verl_src verl 源码目录，容器里用 editable 模式安装

2. 上传文件到服务器

在本地机器执行，按你的服务器地址替换 USER@SERVER。

# 项目最小必需文件：保留 grpodata/verl 这一层目录
rsync -av ./grpodata \
  ./reward_functions.py \
  ./zhenghou_btree.json \
  USER@SERVER:/data/tcm_final/

# 上传你已经 SFT/继续训练好的 Qwen3.5 模型目录
rsync -av /path/to/your/qwen35_tcm_sft/ USER@SERVER:/data/models/qwen35_tcm_sft/
注意：本地路径 ./... 到服务器后不会存在，后面都使用容器内路径 /workspace/tcm、/workspace/model、/workspace/output。

3. 服务器宿主机检查

在服务器上执行：

nvidia-smi
docker --version

mkdir -p /data/tcm_final /data/models/qwen35_tcm_sft /data/outputs/tcm_grpo /data/verl_src

test -f /data/tcm_final/reward_functions.py
test -f /data/tcm_final/zhenghou_btree.json
test -f /data/tcm_final/grpodata/verl/bianzheng_grpo_train.jsonl
test -f /data/tcm_final/grpodata/verl/bianzheng_grpo_test.jsonl

如果你选择把 jsonl 直接放到 /data/tcm_final/verl，后面的 DATA_JSONL_DIR 改成 /workspace/tcm/verl。

4. 拉取并进入 Docker

这里用 vLLM 版官方镜像。若镜像标签后续更新，可以到 DockerHub 的 verlai/verl 选择新的 vLLM 标签。

export IMAGE=verlai/verl:vllm011.latest
docker pull "${IMAGE}"

export PROJECT_DIR=/data/tcm_final
export MODEL_DIR=/data/models/qwen35_tcm_sft
export OUTPUT_DIR=/data/outputs/tcm_grpo
export VERL_SRC_DIR=/data/verl_src

docker run --rm -it \
  --name tcm-verl-grpo \
  --runtime=nvidia \
  --gpus all \
  --net=host \
  --ipc=host \
  --shm-size=128g \
  --ulimit memlock=-1 \
  --ulimit stack=67108864 \
  --cap-add=SYS_ADMIN \
  -v "${PROJECT_DIR}:/workspace/tcm" \
  -v "${MODEL_DIR}:/workspace/model" \
  -v "${OUTPUT_DIR}:/workspace/output" \
  -v "${VERL_SRC_DIR}:/workspace/verl_src" \
  -v "${HOME}/.cache/huggingface:/root/.cache/huggingface" \
  -w /workspace \
  "${IMAGE}" \
  bash
进入容器后，后续所有路径都用容器内路径：模型是 /workspace/model，项目是 /workspace/tcm，输出是 /workspace/output。

5. 容器内安装 verl 源码

官方镜像带了大部分训练/推理依赖，但通常仍需要安装 verl 源码本身。

cd /workspace/verl_src

fi

cd /workspace/verl_src/verl

6. 把 jsonl 转成 parquet

你的数据字段已经符合 verl 常见格式：data_source、prompt、ability、reward_model、extra_info。

cd /workspace

# 如果你的服务器上是 /data/tcm_final/grpodata/verl，则这里保持默认。
# 如果你按第 3 步示例把文件直接放在 /data/tcm_final/verl，则改成 /workspace/tcm/verl。
export DATA_JSONL_DIR=/workspace/tcm/grpodata/verl
export DATA_PARQUET_DIR=/workspace/tcm/verl_data

python - <<'PY'
import json
import os
from pathlib import Path
import pandas as pd

src = Path(os.environ.get("DATA_JSONL_DIR", "/workspace/tcm/grpodata/verl"))
out = Path(os.environ.get("DATA_PARQUET_DIR", "/workspace/tcm/verl_data"))
out.mkdir(parents=True, exist_ok=True)

for split in ["train", "test"]:
    in_file = src / f"bianzheng_grpo_{split}.jsonl"
    rows = []
    with in_file.open("r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if line:
                rows.append(json.loads(line))
    df = pd.DataFrame(rows)
    out_file = out / f"bianzheng_grpo_{split}.parquet"
    df.to_parquet(out_file, index=False)
    print(split, len(df), "->", out_file)
PY

python - <<'PY'
import pandas as pd
for split in ["train", "test"]:
    p = f"/workspace/tcm/verl_data/bianzheng_grpo_{split}.parquet"
    df = pd.read_parquet(p)
    print(p, len(df), df.columns.tolist())
    print(df.iloc[0]["reward_model"])
PY
如果你把数据放在不同目录，记得同步修改 Python 里的 src = Path(...)。

7. 先跑 smoke GRPO

smoke 只跑很小 batch，目的是验证：模型能加载、parquet 能读、奖励函数能调用、vLLM rollout 能启动、FSDP 能更新。

cd /workspace/verl_src/verl

export HYDRA_FULL_ERROR=1
export TOKENIZERS_PARALLELISM=false
export VLLM_USE_V1=1
export RAY_DEDUP_LOGS=0
export WANDB_MODE=offline

python -m verl.trainer.main_ppo \
  data.train_files=/workspace/tcm/verl_data/bianzheng_grpo_train.parquet \
  data.val_files=/workspace/tcm/verl_data/bianzheng_grpo_test.parquet \
  data.train_batch_size=8 \
  data.max_prompt_length=1024 \
  data.max_response_length=512 \
  data.filter_overlong_prompts=True \
  data.truncation=error \
  data.shuffle=False \
  algorithm.adv_estimator=grpo \
  algorithm.use_kl_in_reward=False \
  custom_reward_function.path=/workspace/tcm/reward_functions.py \
  custom_reward_function.name=compute_score \
  reward_model.reward_manager=naive \
  actor_rollout_ref.model.path=/workspace/model \
  actor_rollout_ref.model.trust_remote_code=True \
  actor_rollout_ref.model.use_remove_padding=True \
  actor_rollout_ref.model.enable_gradient_checkpointing=True \
  actor_rollout_ref.actor.strategy=fsdp \
  actor_rollout_ref.actor.optim.lr=5e-7 \
  actor_rollout_ref.actor.ppo_mini_batch_size=4 \
  actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=1 \
  actor_rollout_ref.actor.use_kl_loss=False \
  actor_rollout_ref.actor.kl_loss_coef=0.0 \
  actor_rollout_ref.actor.entropy_coeff=0 \
  actor_rollout_ref.actor.use_torch_compile=False \
  actor_rollout_ref.actor.fsdp_config.param_offload=False \
  actor_rollout_ref.actor.fsdp_config.optimizer_offload=True \
  actor_rollout_ref.ref.strategy=fsdp \
  actor_rollout_ref.ref.log_prob_micro_batch_size_per_gpu=1 \
  actor_rollout_ref.ref.fsdp_config.param_offload=True \
  actor_rollout_ref.rollout.name=vllm \
  actor_rollout_ref.rollout.mode=sync \
  actor_rollout_ref.rollout.n=1 \
  actor_rollout_ref.rollout.tensor_model_parallel_size=1 \
  actor_rollout_ref.rollout.gpu_memory_utilization=0.40 \
  trainer.project_name=tcm-grpo \
  trainer.experiment_name=qwen35_grpo_smoke \
  trainer.logger='["console"]' \
  trainer.nnodes=1 \
  trainer.n_gpus_per_node=1 \
  trainer.default_local_dir=/workspace/output/qwen35_grpo_smoke \
  trainer.critic_warmup=0 \
  trainer.val_before_train=False \
  trainer.save_freq=-1 \
  trainer.test_freq=1 \
  trainer.total_epochs=1
smoke 默认 1 张 GPU。如果你的模型 1 卡放不下，把 trainer.n_gpus_per_node 改成可用 GPU 数， 并把 actor_rollout_ref.rollout.tensor_model_parallel_size 改成 2 或 4。

9. 显存不够时怎么改

现象优先调整
vLLM 初始化 OOM 降低 actor_rollout_ref.rollout.gpu_memory_utilization 到 0.35；降低 MAX_RESPONSE_LENGTH；增大 ROLLOUT_TP
actor update OOM 保持 ppo_micro_batch_size_per_gpu=1；开启/保留 optimizer offload；降低 PPO_MINI_BATCH_SIZE
吞吐太慢 smoke 过后把 ROLLOUT_N 从 1 加到 4 或 8；逐步增加 TRAIN_BATCH_SIZE
模型太大，单卡放不下 rollout 把 ROLLOUT_TP 设为 2、4 或 8，且总 GPU 数要能被 TP 合理划分

10. 常见报错

找不到 zhenghou_btree.json

reward_functions.py 用 os.path.dirname(__file__) 读取同目录文件，所以必须保证容器内：

ls /workspace/tcm/reward_functions.py /workspace/tcm/zhenghou_btree.json

找不到 parquet

重新确认转换输出：

ls -lh /workspace/tcm/verl_data/*.parquet

vLLM 或 torch 版本冲突

不要在官方镜像里随手 pip install -U torch vllm transformers。先使用镜像自带版本，只对 verl 执行 pip install --no-deps -e .。

多模态 Qwen3.5 类名兼容问题

如果报 AutoModelForVision2Seq 相关错误，先检查 transformers 版本和模型 config 的 architectures。临时绕过可在 smoke 阶段加：

+actor_rollout_ref.model.override_config._attn_implementation=eager

如果仍报错，需要根据模型实际 config 做一处 verl/transformers 兼容 patch。
