# 通用设计规则

## 输入假设

输入是一个 JSON 列表，每个元素可能包含：

- `type`: `text` / `image` / `header` / `page_number`
- `id`: 唯一标识
- `text`: 文本内容
- `img_path`: 图片路径
- `image_caption`: 图片标题或图注

## 核心约束

1. 只围绕 `type="image"` 的图片生成 VQA。
2. 每个 QA 只绑定一张图片，不允许把多张图的描述混到同一答案里。
3. 图文可以跨页关联，允许结合相邻页面、同节标题、图号附近文本综合判断。
4. 如果只有标题、没有图片或没有可支撑答案的文字，不要编造内容。
5. 当信息不足时，输出 `<empty></empty>`，不要输出“无法生成”“请补充数据”之类自然语言。
6. 问题必须是“看图可答 + 结合文本可证实”的类型，不要生成脱离图像的纯常识题。
7. 答案必须尽量忠实原文，不扩写未经文本支持的诊断或治疗结论。
8. 若图注仅有编号、没有内容，应优先向前后文本检索该图对应描述。

## 题型建议

- 观察题: 问图中可见征象、颜色、形态、位置、波形、纹理
- 识别题: 问该图对应何种面象、色象、脉象或证候线索
- 辨证题: 问该征象提示的病机、主病、寒热虚实倾向
- 临床题: 问其临床意义、治疗原则或诊断提示

## 不应生成的题

- 答案无法被图文共同支撑的问题
- 需要外部知识才能回答的问题
- 指代模糊的问题，如“这说明了什么”
- 把“图中没有给出”的内容写成确定事实
- 把章节标题硬写成答案

## 输出格式

统一输出 XML：

```xml
<chapter>
<title>章节标题ID</title>
<qa_pair>
<label>图片编号</label>
<question>问题文本</question>
<answer>答案文本</answer>
</qa_pair>
</chapter>
```

## 输出清洗要求

- 禁止输出思维链
- 禁止输出 markdown 代码块
- 禁止输出额外解释
- 最终输出只能是 XML 标签内容或 `<empty></empty>`
